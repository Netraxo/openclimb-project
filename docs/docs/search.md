# OpenClimb Search

## Purpose

This document defines how users search and filter climbing content in OpenClimb.

The goal is to help climbers quickly find the best climbing destination based on their preferences.

---

# Search Scope

Users can search for:

- Areas
- Crags
- Sectors
- Routes

---

# Search by Name

Users can search by name.

Examples:

- Góry Kaczawskie
- Złoty Las
- Sector A
- Jack Sparrow

---

# Search by Grade

Users can search routes by:

- Exact grade
- Grade range

Examples:

- VI.2
- VI.2 – VI.3
- V – VI
- VI.3+

The search supports both official and community grades.

---

# Search by Climbing Type

Users can filter routes by:

- Sport
- Trad

Future support:

- Boulder
- Indoor
- Ice
- Dry Tooling

---

# Search Filters

Available filters:

- Grade
- Climbing type
- Number of quickdraws
- Maximum route height
- Approach time
- Sun exposure
- Family friendly
- Classic routes
- User rating
- Routes with warnings

---

# Sorting

Results can be sorted by:

- Best rated
- Closest
- Easiest
- Hardest
- Shortest approach
- Number of classic routes

---

# Map Integration

The map is the primary navigation method.

Search results are synchronized with the map.

Zoom levels:

Area → Crag → Sector → Route

Users can open a sector directly by zooming in on the map.

---

# Design Principles

- Search should require as few clicks as possible.
- The map is always the primary navigation tool.
- Filters must be simple and fast.
- Users should find suitable routes within seconds.
